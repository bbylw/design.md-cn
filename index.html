概述

建筑极简主义与新闻庄重感相结合

UI 唤起一种高级哑光质感——如同高端大报或当代画廊。DESIGN.md 文件将机器可读的设计令牌（YAML 前置事项）与人类可读的设计原理（Markdown 正文）相结合。

令牌提供精确值，正文解释这些值存在的原因以及如何应用它们。阅读此文件的智能体将生成一个符合设计意图的 UI。

文件结构

两层架构设计

1

YAML 前置事项

机器可读的设计令牌，以文件顶部的 --- 围栏分隔。

2

Markdown 正文

人类可读的设计原理，按 ## 章节组织。

令牌模式

结构化设计令牌

令牌是规范值，正文提供如何应用它们的上下文。

DESIGN.md 

---
name: Heritage
colors:
  primary: "#1A1C1E"
  secondary: "#6C7278"
  tertiary: "#B8422E"
  neutral: "#F7F5F2"
typography:
  h1:
    fontFamily: Public Sans
    fontSize: 3rem
  body-md:
    fontFamily: Public Sans
    fontSize: 1rem
  label-caps:
    fontFamily: Space Grotesk
    fontSize: 0.75rem
rounded:
  sm: 4px
  md: 8px
spacing:
  sm: 8px
  md: 16px
---

令牌类型

类型 格式 示例
颜色 # + 十六进制 (sRGB) "#1A1C1E"
尺寸 数字 + 单位 (px, em, rem) 48px, -0.02em
令牌引用 {path.to.token} {colors.primary}
排版 包含字体属性的对象 fontFamily 等
CLI 参考

命令行工具

安装 @google/design.md 即可使用完整的命令行工具集。

lint

验证文件结构

npx @google/design.md lint DESIGN.md

验证 DESIGN.md 文件的结构正确性，检查损坏的令牌引用和 WCAG 对比度比率。

diff

比较版本差异

npx @google/design.md diff DESIGN.md DESIGN-v2.md

比较两个 DESIGN.md 文件，检测令牌级别和正文回归。

export

导出为其他格式

npx @google/design.md export --format tailwind DESIGN.md

将令牌导出为 Tailwind 主题配置或 DTCG tokens.json 格式。

spec

输出格式规范

npx @google/design.md spec --rules

输出 DESIGN.md 格式规范，可用于将规范上下文注入智能体提示。

Linting 规则

七条验证规则

Linter 针对解析后的 DESIGN.md 运行七条规则，每条规则产生固定严重级别的发现结果。

规则 严重级别 检查内容
broken-ref 错误 令牌引用无法解析为任何已定义令牌
missing-primary 警告 颜色已定义但不存在 primary 颜色
contrast-ratio 警告 组件对比度低于 WCAG AA 最低标准 (4.5:1)
orphaned-tokens 警告 颜色令牌已定义但未被任何组件引用
token-summary 信息 每个章节中定义的令牌数量摘要
missing-sections 信息 可选章节（间距、圆角）缺失
missing-typography 警告 颜色已定义但不存在排版令牌
section-order 警告 章节出现在规范顺序之外
